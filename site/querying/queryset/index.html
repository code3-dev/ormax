    Querying Data with Ormax¶

    Ormax provides a powerful, chainable QuerySet API for building database queries. This guide will show you how to effectively query your data using Ormax's intuitive syntax.

    Understanding QuerySet¶

    A QuerySet represents a collection of objects from your database. It has two key characteristics:

    1. Lazy evaluation: Queries aren't executed until you actually need the data
    2. Chainable: You can keep adding filters and operations to refine your query
    # This doesn't hit the database yet
books = Book.objects().filter(pages__gt=300).order_by("-published_date")

# This executes the query
await books.all()

    Basic Query Operations¶

    Retrieving All Objects¶

    # Get all books
all_books = await Book.objects().all()

# Get the first book
first_book = await Book.objects().first()

# Get the last book (by primary key)
last_book = await Book.objects().last()

    Getting a Single Object¶

    # Get a book by primary key
book = await Book.objects().get(id=1)

# Get by other fields
author = await Author.objects().get(name="احمد محمودی")

# Using get() with filters
book = await Book.objects().get(title="شیر و خورشید", pages=350)

    Note: get() will raise DoesNotExist if no object is found, and DatabaseError if multiple objects match.

    Filtering Results¶

    # Exact match
books = await Book.objects().filter(title="شیر و خورشید").all()

# Multiple filters (AND condition)
books = await Book.objects().filter(
    title="شیر و خورشید", 
    pages=350
).all()

# Chained filters
books = await Book.objects().filter(pages__gt=300).filter(title__contains="sun").all()

    Excluding Results¶

    # Exclude books with less than 200 pages
books = await Book.objects().exclude(pages__lt=200).all()

# Combine filter and exclude
books = await Book.objects().filter(pages__gt=100).exclude(title__startswith="A").all()

    Field Lookups¶

    Ormax supports various field lookups to create more specific queries:

    Basic Lookups¶

    Lookup Description Example
    __exact Exact match (default) title__exact="شیر و خورشید"
    __iexact Case-insensitive exact match title__iexact="shir va khorshid"
    __contains Contains the value title__contains="sun"
    __icontains Case-insensitive contains title__icontains="sun"
    __gt Greater than pages__gt=300
    __gte Greater than or equal to pages__gte=300
    __lt Less than pages__lt=100
    __lte Less than or equal to pages__lte=100
    __in In a list of values pages__in=[100, 200, 300]
    __isnull Is null published_date__isnull=True 
    # Books with 100-300 pages
books = await Book.objects().filter(
    pages__gte=100, 
    pages__lte=300
).all()

# Books with specific page counts
books = await Book.objects().filter(pages__in=[100, 200, 300]).all()

# Books with titles containing "sun" (case-insensitive)
books = await Book.objects().filter(title__icontains="sun").all()

    Date and Time Lookups¶

    For DateTimeField and DateField:

    Lookup Description Example
    __year Match year published_date__year=2020
    __month Match month published_date__month=5
    __day Match day published_date__day=15
    __week_day Match weekday (1=Sunday) published_date__week_day=1
    __hour Match hour created_at__hour=12
    __minute Match minute created_at__minute=30
    __second Match second created_at__second=45 
    # Books published in 2020
books = await Book.objects().filter(published_date__year=2020).all()

# Books published in May
books = await Book.objects().filter(published_date__month=5).all()

    Ordering Results¶

    Basic Ordering¶

    # Order by title (ascending)
books = await Book.objects().order_by("title").all()

# Order by pages descending
books = await Book.objects().order_by("-pages").all()

# Multiple ordering
books = await Book.objects().order_by("-published_date", "title").all()

    Using Expressions in Ordering¶

    # Order by calculated field
books = await Book.objects().annotate(
    page_ratio="pages / 10"
).order_by("page_ratio").all()

    Limiting and Pagination¶

    Limiting Results¶

    # Get first 10 books
books = await Book.objects().limit(10).all()

# Get books 11-20 (for pagination)
books = await Book.objects().limit(10).offset(10).all()

    Pagination Helper¶

    Ormax includes a utility function for pagination:

    from ormax.utils import paginate_list

# Get page 2 with 10 items per page
books = await Book.objects().all()
paginated_books, total_pages, total_items = paginate_list(books, page=2, per_page=10)

    Aggregation and Statistics¶

    Counting Records¶

    # Count all books
total = await Book.objects().count()

# Count filtered books
long_books = await Book.objects().filter(pages__gt=300).count()

    Checking Existence¶

    # Check if any books exist
exists = await Book.objects().exists()

# Check if specific book exists
exists = await Book.objects().filter(title="شیر و خورشید").exists()

    Advanced Aggregation¶

    from ormax.query import Aggregation

# Get average number of pages
avg_pages = await Aggregation.avg(Book.objects(), "pages")

# Get total pages
total_pages = await Aggregation.sum(Book.objects(), "pages")

# Get minimum/maximum
min_pages = await Aggregation.min(Book.objects(), "pages")
max_pages = await Aggregation.max(Book.objects(), "pages")

    Advanced Query Operations¶

    Distinct Results¶

    # Get distinct authors
authors = await Book.objects().distinct().values_list("author", flat=True)

    Grouping Results¶

    # Group by author and count books
author_counts = await Book.objects().values("author").annotate(count="COUNT(*)").all()

# Group by publication year
year_counts = await Book.objects().annotate(
    year="EXTRACT(YEAR FROM published_date)"
).values("year").annotate(count="COUNT(*)").all()

    Using Only Specific Fields¶

    # Load only title and pages
books = await Book.objects().only("title", "pages").all()

# Accessing other fields will return None
for book in books:
    print(book.title)  # Works
    print(book.author)  # Returns None

    Deferring Fields¶

    # Load all fields except bio
authors = await Author.objects().defer("bio").all()

# Accessing bio will trigger an additional query
for author in authors:
    print(author.name)  # Works
    print(author.bio)   # Triggers additional query

    Using Specific Indexes¶

    # Force using a specific index
books = await Book.objects().using_index("title_idx").filter(title__startswith="A").all()

    Locking Records¶

    # Lock records for update
book = await Book.objects().filter(id=1).for_update().first()
# This record is now locked until the transaction ends
book.pages += 1
await book.save()

    Query Timeout¶

    # Set a timeout for the query
books = await Book.objects().timeout(5).filter(pages__gt=500).all()
# Will raise TimeoutError if query takes more than 5 seconds

    Raw SQL Queries¶

    For complex queries that can't be expressed with the QuerySet API:

    Executing Raw Queries¶

    # Execute raw SQL
results = await Book.objects().raw(
    "SELECT * FROM book WHERE pages > ?", 
    (300,)
).execute()

# Get first result
result = await Book.objects().raw(
    "SELECT * FROM book WHERE id = ?", 
    (1,)
).fetch_one()

    Async Iteration¶

    # Async iteration over results
async for row in Book.objects().raw("SELECT * FROM book"):
    print(row)

    Mapping Results to Models¶

    # Map raw results to model instances
books = await Book.objects().raw(
    "SELECT * FROM book WHERE pages > ?",
    (300,)
).map_to_model()

    Query Optimization¶

    Select Related (JOIN)¶

    Use select_related to follow foreign key relationships in a single query:

    # Get books with authors in one query
books = await Book.objects().select_related("author").all()

# Now accessing book.author won't trigger additional queries
for book in books:
    print(book.author.name)  # No additional query

    Prefetch Related (Separate Queries)¶

    Use prefetch_related to fetch related objects in separate queries:

    # Get authors with their books
authors = await Author.objects().prefetch_related("books").all()

# Now accessing author.books.all() won't trigger additional queries per author
for author in authors:
    for book in author.books.all():  # No additional queries
        print(book.title)

    Combining Both Techniques¶

    # Complex query with both select_related and prefetch_related
books = await Book.objects().select_related("author").prefetch_related("reviews").all()

    Common Query Patterns¶

    Getting the Earliest/Latest Records¶

    # Get earliest book (by primary key or created_at)
earliest = await Book.objects().earliest()

# Get latest book
latest = await Book.objects().latest()

# Specify field to use
latest = await Book.objects().latest("published_date")

    In Bulk Operations¶

    # Get multiple objects by ID
books = await Book.objects().in_bulk([1, 2, 3])

# Get by custom field
authors = await Author.objects().in_bulk(
    ["ahmad", "saman"], 
    field_name="username"
)

    Updating Multiple Records¶

    # Update all books with less than 200 pages
await Book.objects().filter(pages__lt=200).update(
    pages=200
)

    Deleting Multiple Records¶

    # Delete all books with less than 100 pages
await Book.objects().filter(pages__lt=100).delete()

    Performance Considerations¶

    Avoiding N+1 Query Problem¶

    The N+1 query problem occurs when you trigger a new query for each item in a loop:

    # BAD: N+1 queries
books = await Book.objects().all()
for book in books:
    print(book.author.name)  # Triggers a query for each book

# GOOD: Use select_related
books = await Book.objects().select_related("author").all()
for book in books:
    print(book.author.name)  # No additional queries

    Using Values and Values List¶

    When you only need specific fields, use values() or values_list():

    # Get only titles and pages as dictionaries
book_data = await Book.objects().values("title", "pages").all()

# Get only titles as a flat list
titles = await Book.objects().values_list("title", flat=True).all()

    Batch Processing¶

    For large result sets, process in batches:

    async def process_batch(batch):
    for book in batch:
        # Process book
        await book.update(pages=book.pages + 1)

# Process books in batches of 100
async for batch in Book.objects().batch(100):
    await process_batch(batch)

    Complete Examples¶

    Book Search Example¶

    async def search_books(
    title=None, 
    min_pages=None, 
    max_pages=None, 
    author_name=None,
    page=1,
    per_page=10
):
    """Search books with various filters"""
    qs = Book.objects()

    # Apply filters
    if title:
        qs = qs.filter(title__icontains=title)
    if min_pages:
        qs = qs.filter(pages__gte=min_pages)
    if max_pages:
        qs = qs.filter(pages__lte=max_pages)
    if author_name:
        qs = qs.filter(author__name__icontains=author_name)

    # Apply ordering
    qs = qs.order_by("-published_date", "title")

    # Apply pagination
    total = await qs.count()
    books = await qs.limit(per_page).offset((page-1)*per_page).all()

    # Get authors with prefetch_related to avoid N+1
    author_ids = list(set(book.author_id for book in books))
    authors = await Author.objects().filter(id__in=author_ids).all()
    author_map = {author.id: author for author in authors}

    # Add authors to books
    for book in books:
        book.author = author_map.get(book.author_id)

    return {
        "books": books,
        "total": total,
        "page": page,
        "pages": (total + per_page - 1) // per_page
    }

    Dashboard Statistics Example¶

    async def get_dashboard_stats():
    """Get statistics for dashboard"""
    # Books stats
    total_books = await Book.objects().count()
    average_pages = await Aggregation.avg(Book.objects(), "pages")

    # Authors stats
    total_authors = await Author.objects().count()
    authors_with_books = await Author.objects().filter(books__isnull=False).count()

    # Books by year
    books_by_year = await Book.objects().annotate(
        year="EXTRACT(YEAR FROM published_date)"
    ).values("year").annotate(count="COUNT(*)").order_by("year").all()

    # Popular authors
    popular_authors = await Author.objects().annotate(
        book_count="COUNT(books)"
    ).order_by("-book_count").limit(5).all()

    return {
        "total_books": total_books,
        "average_pages": average_pages,
        "total_authors": total_authors,
        "authors_with_books": authors_with_books,
        "books_by_year": books_by_year,
        "popular_authors": popular_authors
    }

    Troubleshooting Common Issues¶

    Empty QuerySet¶

    Issue: await Book.objects().filter(title="test").all() returns empty list

    Solution: - Check if records actually exist in the database - Verify the filter condition matches your data - Check for case sensitivity (use __icontains for case-insensitive matching)

    MultipleObjectsReturned¶

    Issue: await Book.objects().get(title="test") raises DatabaseError: Multiple objects returned

    Solution: - Use filter() instead of get() when expecting multiple results - Add more specific filters to ensure only one object matches - Use first() if you only need one result: await Book.objects().filter(title="test").first()

    FieldDoesNotExist¶

    Issue: DatabaseError: Field 'xyz' does not exist on model 'Book'

    Solution: - Check that the field name is correctly spelled - Verify the field is defined in your model - For related fields, ensure the relationship is properly defined

    Performance Problems¶

    Issue: Queries are slow with large datasets

    Solution: - Add indexes to frequently queried fields - Use select_related or prefetch_related to avoid N+1 queries - Use only() or defer() to load only necessary fields - Limit the number of results with limit()

    Best Practices¶

    Use Asynchronous Pattern Correctly¶

    # CORRECT
books = await Book.objects().filter(pages__gt=300).all()

# INCORRECT (will return a coroutine object, not results)
books = Book.objects().filter(pages__gt=300).all()

    Chain Queries for Readability¶

    # GOOD
books = await Book.objects() \
    .filter(pages__gt=300) \
    .order_by("-published_date") \
    .limit(10) \
    .all()

# BAD (harder to read and modify)
books = await Book.objects().filter(pages__gt=300).order_by("-published_date").limit(10).all()

    Use Transactions for Related Operations¶

    async with db.transaction():
    # Create book
    book = await Book.create(title="New Book", pages=200)

    # Update author's book count
    author = await Author.get(id=book.author_id)
    author.book_count += 1
    await author.save()
