    Ormax ORM: Modern Async Python ORM¶

    Ormax is a lightweight, high-performance asynchronous Object-Relational Mapping (ORM) library for Python that provides an intuitive interface for database operations. Designed for modern Python applications, Ormax supports multiple database backends while maintaining a clean, Pythonic API with full async/await support.

    Why Ormax?¶

    Ormax bridges the gap between simplicity and power in Python ORM solutions. Unlike heavier alternatives, Ormax offers:

    • ✅ True async support - Built from the ground up for asyncio
    • ✅ Multiple database compatibility - Works with SQLite, PostgreSQL, MySQL, MSSQL, Oracle, and Aurora
    • ✅ Minimal overhead - Optimized for performance with __slots__ and efficient query building
    • ✅ Intuitive API - Chainable QuerySet interface familiar to Django ORM users
    • ✅ Production-ready - Includes transaction management, connection pooling, and robust error handling

    Quick Start¶

    Installation¶

    pip install ormax

    Basic Usage¶

    import ormax
from ormax.fields import CharField, IntegerField

# Connect to database
db = ormax.Database("sqlite:///mydatabase.db")

# Define a model
class Book(ormax.Model):
    title = CharField(max_length=200)
    pages = IntegerField()

# Register model and create tables
db.register_model(Book)
await db.create_tables()

# Create a book
book = await Book.create(title="شیر و خورشید", pages=350)

# Query books
all_books = await Book.objects().all()
ahmad_books = await Book.objects().filter(title__contains="احمد").all()

    Key Features¶

    🚀 Async/Await Support¶

    Fully asynchronous design for high-performance applications with proper connection pooling.

    🗄️ Multiple Database Support¶

    Works seamlessly with: - SQLite - PostgreSQL - MySQL - Microsoft SQL Server - Oracle - Amazon Aurora

    🔗 Relationship Management¶

    Complete support for ForeignKey relationships with reverse lookups and efficient prefetching.

    💰 Transactions¶

    Nested transactions with savepoint support for complex business logic.

    📊 Query Optimization¶

    Chainable QuerySet API with: - select_related and prefetch_related to avoid N+1 queries - defer and only for field optimization - Index hinting with using_index
