    Debugging Techniques for Ormax Applications¶

    Debugging database-driven applications requires specialized techniques to understand what's happening under the hood. This guide covers comprehensive debugging strategies specifically designed for Ormax-based applications.

    Understanding Ormax Error Types¶

    Before debugging, it's important to understand the different error types Ormax can raise:

    Common Error Categories¶

    Error Type When It Occurs Typical Causes
    DatabaseError General database issues Connection problems, syntax errors
    ValidationError Data validation fails Invalid field values, constraint violations
    DoesNotExist Record not found get() called with no matching records
    MultipleObjectsReturned Multiple records found get() called with multiple matches
    IntegrityError Database integrity violation Foreign key constraints, unique constraints
    TransactionRollbackError Transaction failure Deadlocks, serialization failures

    Decoding Error Messages¶

    Ormax error messages contain valuable information:

    DatabaseError: (sqlite3.IntegrityError) UNIQUE constraint failed: books.title
[SQL: INSERT INTO books (title, pages) VALUES (?, ?)]
[parameters: ('Duplicate Title', 200)]

    This tells us: - The error type (DatabaseError) - The underlying database error (sqlite3.IntegrityError) - The specific constraint that failed (UNIQUE constraint failed: books.title) - The SQL query that caused the error - The parameters used in the query

    Debugging Query Issues¶

    Viewing Executed Queries¶

    The most fundamental debugging technique is seeing what SQL is being executed:

    Method 1: Enable Debug Logging¶

    import logging

# Configure logging to show queries
logging.basicConfig(level=logging.DEBUG)
logger = logging.getLogger("ormax")
logger.setLevel(logging.DEBUG)

    This will log all queries to the console:

    DEBUG:ormax:Executing query: SELECT * FROM books WHERE pages > ?
DEBUG:ormax:Parameters: (300,)

    Method 2: Using the explain() Method¶

    Get database-specific execution plans:

    # Get query execution plan
plan = await Book.objects().filter(pages__gt=300).explain()
print(plan)

# For more detailed analysis (PostgreSQL)
detailed_plan = await Book.objects().filter(pages__gt=300).explain(analyze=True)

    Method 3: Manual Query Inspection¶

    # Get the raw SQL query
query, params = Book.objects().filter(pages__gt=300)._build_select_query()
print(f"SQL: {query}")
print(f"Parameters: {params}")

    Common Query Issues and Solutions¶

    Issue: Incorrect WHERE Clause¶

    Symptoms: - Unexpected results - Missing records - Extra records

    Diagnosis: - Check the generated WHERE clause - Verify parameter values

    Solution: 

    # Check the query
qs = Book.objects().filter(title__contains="sun", pages__gt=300)
print(qs._build_select_query())

# Fix by using correct field lookups
qs = Book.objects().filter(title__icontains="sun", pages__gt=300)

    Issue: N+1 Query Problem¶

    Symptoms: - Many similar queries with different parameters - Performance degrades with larger datasets

    Diagnosis: - Look for repeated queries in logs - Check if relationship fields are being accessed in loops

    Solution: 

    # BAD: N+1 queries
books = await Book.objects().all()
for book in books:
    print(book.author.name)  # Triggers a query per book

# GOOD: Use select_related
books = await Book.objects().select_related("author").all()
for book in books:
    print(book.author.name)  # No additional queries

    Issue: Incorrect JOINs¶

    Symptoms: - Missing related data - Duplicate records - Incorrect results with prefetch_related

    Diagnosis: - Check the generated JOIN clauses - Verify relationship definitions

    Solution: 

    # Check the query
qs = Book.objects().select_related("author")
print(qs._build_select_query())

# Verify relationship definition
class Book(Model):
    author = ForeignKeyField(Author, related_name="books")

    Common Connection Issues and Solutions¶

    Issue: Connection Timeout¶

    Symptoms: - OperationalError: Connection timeout - Application hangs during database operations

    Solutions: - Increase connection timeout: Database(url, connect_timeout=15.0) - Check network connectivity - Verify database server is running - Reduce application load

    Issue: Too Many Connections¶

    Symptoms: - OperationalError: Too many connections - Application fails to connect during peak times

    Solutions: - Increase database connection limit - Optimize connection pool settings: 

    db = Database(url, min_size=5, max_size=25)
    - Ensure connections are properly closed - Implement connection recycling

    Issue: Authentication Failure¶

    Symptoms: - OperationalError: Authentication failed - Access denied for user

    Solutions: - Verify username and password - Check database user permissions - Ensure correct database name - Test credentials with database client

    Debugging Relationship Issues¶

    Diagnosing Relationship Problems¶

    Step 1: Verify Relationship Definitions¶

    # Check forward relationship
print(Book._fields['author'])  # Should be a ForeignKeyField

# Check reverse relationship
print(Author._relationships.get_reverse_manager('books'))

    Step 2: Inspect Relationship Data¶

    # Get book with author
book = await Book.objects().get(id=1)

# Check raw foreign key value
print(f"Author ID: {book.author_id}")

# Attempt to access related object
try:
    author = await book.author
    print(f"Author name: {author.name}")
except Exception as e:
    print(f"Error accessing author: {e}")

    Common Relationship Issues and Solutions¶

    Issue: Foreign Key Constraint Violation¶

    Symptoms: - IntegrityError: FOREIGN KEY constraint failed - Cannot save model with relationship

    Diagnosis: - Check if related object exists - Verify primary key values - Check database schema

    Solution: 

    # Verify related object exists
author = await Author.get(id=author_id)
if not author:
    print(f"Author with ID {author_id} does not exist")

# Ensure related object is saved before assignment
new_author = Author(name="New Author")
await new_author.save()  # Save first
book.author = new_author  # Then assign

    Issue: Accessing Relationship Returns None¶

    Symptoms: - await book.author returns None when it shouldn't - Relationship appears broken

    Diagnosis: - Check if foreign key field is None - Verify relationship was properly assigned

    Solution: 

    # Check foreign key value
print(book.author_id)  # If None, relationship isn't set

# Properly assign relationship
author = await Author.get(id=1)
book.author = author
await book.save()

    Issue: Reverse Relationship Not Working¶

    Symptoms: - await author.books.all() returns empty list - Reverse relationship seems broken

    Diagnosis: - Check relationship name - Verify related_name is correct

    Solution: 

    # Check how the relationship is defined
class Book(Model):
    author = ForeignKeyField(Author, related_name="books")

# Correct usage
books = await author.books.all()

# If related_name wasn't specified
books = await author.book_set.all()

    Debugging Transactions¶

    Diagnosing Transaction Issues¶

    Step 1: Verify Transaction State¶

    def print_transaction_status():
    """Print current transaction status"""
    print(f"Current transaction: {db._current_transaction}")
    print(f"Transaction level: {getattr(db, '_transaction_level', 0)}")

    Step 2: Monitor Transaction Execution¶

    async def monitored_transaction():
    print("Starting transaction...")
    try:
        async with db.transaction():
            print("Transaction active")
            # Perform operations
            await Book.create(title="Transaction Book", pages=200)
            print("Book created")
        print("Transaction committed")
    except Exception as e:
        print(f"Transaction rolled back: {e}")
        raise

    Common Transaction Issues and Solutions¶

    Issue: Transaction Not Rolling Back¶

    Symptoms: - Changes persist after exception - Data inconsistency

    Diagnosis: - Verify exception is not being caught and ignored - Check if transaction context is properly exited

    Solution: 

    # CORRECT: Transaction rolls back on exception
async with db.transaction():
    await Book.create(title="Test", pages=100)
    raise ValueError("Force rollback")

# INCORRECT: Exception caught, transaction not rolled back
try:
    async with db.transaction():
        await Book.create(title="Test", pages=100)
        raise ValueError("Force rollback")
except ValueError:
    pass  # Transaction still active!

    Issue: Deadlocks¶

    Symptoms: - TransactionRollbackError: deadlock detected - Intermittent failures

    Diagnosis: - Check query execution order - Monitor lock contention

    Solution: 

    # Always access tables in consistent order
async with db.transaction():
    # Always access authors before books
    author = await Author.get(id=1)
    book = await Book.get(id=1)

    # Update in consistent order
    await author.update(...)
    await book.update(...)

    Issue: Nested Transaction Problems¶

    Symptoms: - Unexpected rollbacks - Data not committed

    Diagnosis: - Check savepoint usage - Verify database supports nested transactions

    Solution: 

    async with db.transaction():
    await Book.create(title="Outer", pages=100)

    try:
        async with db.transaction():
            await Book.create(title="Inner", pages=200)
            raise ValueError("Force inner rollback")
    except ValueError:
        pass  # Inner transaction rolled back, outer continues

    await Book.create(title="Another Outer", pages=300)

    PostgreSQL Debugging¶

    async def debug_postgresql():
    """Get PostgreSQL-specific debug information"""
    # Active queries
    active_queries = await db.connection.fetch_all("""
        SELECT pid, now() - query_start AS duration, query 
        FROM pg_stat_activity 
        WHERE state = 'active'
    """)

    # Locks
    locks = await db.connection.fetch_all("""
        SELECT * FROM pg_locks WHERE granted = false
    """)

    # Replication status
    replication = await db.connection.fetch_all("""
        SELECT * FROM pg_stat_replication
    """)

    return {
        "active_queries": active_queries,
        "locks": locks,
        "replication": replication
    }

        "connection_stats": connection_stats
    }

    import tracemalloc
from ormax.utils import Timer

async def debug_memory_usage():
    """Track memory usage during database operations"""
    tracemalloc.start()

    # Take initial snapshot

    Identifying Memory Leaks¶

    async def check_for_memory_leaks():
    """Check for memory leaks in database operations"""
    import gc

    # Run garbage collection
    gc.collect()

    # Get initial memory usage
    initial = tracemalloc.take_snapshot()

    # Perform operations multiple times
    for _ in range(10):
        books = await Book.objects().all()
        # Process books
        [book.title for book in books]

    # Run garbage collection again
    gc.collect()

    # Get final memory usage
    final = tracemalloc.take_snapshot()

    # Compare snapshots
    stats = final.compare_to(initial, 'lineno')

    # Check for significant memory growth
    if stats and stats[0].size_diff > 1024 * 1024:  # 1MB
        print("Potential memory leak detected:")
        for stat in stats[:5]:
            print(f"  {stat}")
    else:
        print("No significant memory leak detected")

    async def check_migration_status():
    """Check current migration status"""
    # Check if migrations table exists
    exists = await db.connection.fetch_val(

    Step 2: Inspect Migration Scripts¶

    def inspect_migration_script(migration_name):
    """Inspect a migration script without executing it"""
    import importlib.util
    import os

    migration_path = f"migrations/{migration_name}.py"
    if not os.path.exists(migration_path):
        print(f"Migration {migration_name} not found")
        return

    # Load the module without executing
    spec = importlib.util.spec_from_file_location(
        migration_name, migration_path
    )
    module = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(module)

    # Print up and down functions
    print(f"\nMigration: {migration_name}")
    print("Up function:")
    print(inspect.getsource(module.up))
    print("\nDown function:")
    print(inspect.getsource(module.down))

    Common Migration Issues and Solutions¶

    Issue: Migration Already Applied¶

    Symptoms: - DatabaseError: Migration already applied - Duplicate entry errors

    Diagnosis: - Check migrations table - Verify migration script idempotency

    Solution: 

    # Ensure migration is idempotent
async def up(db):
    # Check if column already exists
    columns = await db.connection.fetch_all(
        "PRAGMA table_info(books)"
    )
    if "pages" not in [col["name"] for col in columns]:
        await db.connection.execute(
            "ALTER TABLE books ADD COLUMN pages INTEGER NOT NULL DEFAULT 100"
        )

    Issue: Foreign Key Constraint Violation During Migration¶

    Symptoms: - IntegrityError: FOREIGN KEY constraint failed - Migration fails when adding foreign key

    Diagnosis: - Check existing data - Verify referenced values exist

    Solution: 

    async def up(db):
    # 1. Add column without constraint
    await db.connection.execute(
        "ALTER TABLE books ADD COLUMN author_id INTEGER"
    )

    # 2. Update existing data
    await db.connection.execute(
        "UPDATE books SET author_id = 1 WHERE author_id IS NULL"
    )

    # 3. Add foreign key constraint
    await db.connection.execute(
        "ALTER TABLE books ADD CONSTRAINT fk_author "
        "FOREIGN KEY (author_id) REFERENCES authors(id)"
    )

    Issue: Migration Rollback Failure¶

    Symptoms: - DatabaseError: Cannot drop column used in foreign key constraint - Rollback fails after partial migration

    Diagnosis: - Check database-specific limitations - Verify rollback script

    Solution: 

    async def down(db):
    # For PostgreSQL, need to drop constraints first
    if "postgresql" in db.connection_string:
        await db.connection.execute(
            "ALTER TABLE books DROP CONSTRAINT IF EXISTS fk_author"
        )

    # Now drop the column
    await db.connection.execute(
        "ALTER TABLE books DROP COLUMN author_id"
    )

    Debugging an N+1 Query Problem¶

    import logging
from io import StringIO

def detect_n_plus_one():
    """Detect potential N+1 query patterns in logs"""
    # Configure logger to capture queries
    log_stream = StringIO()
    handler = logging.StreamHandler(log_stream)
    logger = logging.getLogger("ormax")
    logger.addHandler(handler)

    try:
        # Run the code that might have N+1 issues
        books = Book.objects().all()
        for book in books:
            await book.author  # This would trigger N+1 queries

        # Analyze logs
        log_content = log_stream.getvalue()
        queries = [line for line in log_content.split('\n') 
                  if "Executing query" in line]

        # Look for similar queries with different parameters
        pattern_counts = {}
        for query in queries:
            # Extract the base query without parameters
            base_query = re.sub(r'\?.*?FROM', ' ? FROM', query)
            base_query = re.sub(r'WHERE .*? = ', 'WHERE ? = ', base_query)

            pattern_counts[base_query] = pattern_counts.get(base_query, 0) + 1

        # Flag potential N+1 queries
        n_plus_one_queries = [
            (pattern, count) for pattern, count in pattern_counts.items() 
            if count > 5  # Threshold for potential N+1
        ]

        if n_plus_one_queries:
            print("Potential N+1 query patterns detected:")
            for pattern, count in n_plus_one_queries:
                print(f"  {count} occurrences of: {pattern}")
            return True
        return False
    finally:
        logger.removeHandler(handler)

# Usage
if detect_n_plus_one():
    print("Consider using select_related or prefetch_related")

    Debugging a Transaction Deadlock¶

    async def debug_deadlock():
    """Debug a transaction deadlock scenario"""
    # Setup test data
    author1 = await Author.create(name="Author 1")
    author2 = await Author.create(name="Author 2")
    book1 = await Book.create(title="Book 1", author=author1)
    book2 = await Book.create(title="Book 2", author=author2)

    # Create two tasks that might deadlock
    async def update_task1():
        async with db.transaction():
            # Update author1
            await Author.get(id=author1.id).update(name="Updated 1")
            # Try to update book2
            await Book.get(id=book2.id).update(title="Updated Book 2")

    async def update_task2():
        async with db.transaction():
            # Update author2
            await Author.get(id=author2.id).update(name="Updated 2")
            # Try to update book1
            await Book.get(id=book1.id).update(title="Updated Book 1")

    # Run tasks concurrently
    try:
        await asyncio.gather(update_task1(), update_task2())
    except Exception as e:
        print(f"Transaction failed: {e}")

        # Get database-specific deadlock information
        if "postgresql" in db.connection_string:
            locks = await db.connection.fetch_all("""
                SELECT 
                    blocked_locks.pid AS blocked_pid,
                    blocking_locks.pid AS blocking_pid,
                    blocked_activity.query AS blocked_query,
                    blocking_activity.query AS blocking_query
                FROM pg_catalog.pg_locks blocked_locks
                JOIN pg_catalog.pg_stat_activity blocked_activity 
                    ON blocked_activity.pid = blocked_locks.pid
                JOIN pg_catalog.pg_locks blocking_locks 
                    ON blocking_locks.locktype = blocked_locks.locktype
                    AND blocking_locks.DATABASE IS NOT DISTINCT FROM blocked_locks.DATABASE
                    AND blocking_locks.relation IS NOT DISTINCT FROM blocked_locks.relation
                    AND blocking_locks.page IS NOT DISTINCT FROM blocked_locks.page
                    AND blocking_locks.tuple IS NOT DISTINCT FROM blocked_locks.tuple
                    AND blocking_locks.virtualxid IS NOT DISTINCT FROM blocked_locks.virtualxid
                    AND blocking_locks.transactionid IS NOT DISTINCT FROM blocked_locks.transactionid
                    AND blocking_locks.classid IS NOT DISTINCT FROM blocked_locks.classid
                    AND blocking_locks.objid IS NOT DISTINCT FROM blocked_locks.objid
                    AND blocking_locks.objsubid IS NOT DISTINCT FROM blocked_locks.objsubid
                    AND blocking_locks.pid != blocked_locks.pid
                JOIN pg_catalog.pg_stat_activity blocking_activity 
                    ON blocking_activity.pid = blocking_locks.pid
                WHERE NOT blocked_locks.GRANTED
            """)
            if locks:
                print("\nDeadlock details:")
                for lock in locks:
                    print(f"Blocked PID {lock['blocked_pid']} by PID {lock['blocking_pid']}")
                    print(f"Blocked query: {lock['blocked_query']}")
                    print(f"Blocking query: {lock['blocking_query']}")

    def print_model_schema(model_class):
    """Print detailed schema information for a model"""
    print(f"\nModel: {model_class.__name__}")
    print(f"Table: {model_class.get_table_name()}")
    print("\nFields:")

    for field_name, field in model_class._fields.items():
        field_type = field.get_sql_type()
        constraints = []

        if field.primary_key:
            constraints.append("PRIMARY KEY")
        if field.auto_increment:
            constraints.append("AUTOINCREMENT")
        if not field.nullable:
            constraints.append("NOT NULL")
        if field.unique:
            constraints.append("UNIQUE")
        if field.default is not None:
            constraints.append(f"DEFAULT {field.get_default_sql()}")

        print(f"  {field_name}: {field_type}")
        if constraints:
            print(f"    {' | '.join(constraints)}")

    # Relationships
    relationships = []
    for name, manager in model_class._relationships.get_all_related_managers().items():
        relationships.append(f"  {name} -> {manager.related_model_class.__name__}")

    for name, manager in model_class._relationships.get_all_reverse_managers().items():
        relationships.append(f"  {name} <- {manager.related_model_class.__name__}")

    if relationships:
        print("\nRelationships:")
        for rel in relationships:
            print(rel)

    Troubleshooting Common Debugging Issues¶

    Issue: Debugger Doesn't Stop at Breakpoints¶

    Symptoms: - Breakpoints are ignored - Debugger skips over async code

    Solutions: - Ensure "Asyncio Support" is enabled in your IDE - Use asyncio.run() for top-level async code - Set breakpoints before async operations - Check if code is being executed in a different event loop

    Issue: Logging Doesn't Show Queries¶

    Symptoms: - No query logs appear - Debug logs are missing

    Solutions: - Verify logger level is set to DEBUG - Check if logger is properly configured - Ensure you're using the correct logger name ("ormax") - Check if logs are being captured by another handler

    Issue: Memory Profiling Shows False Positives¶

    Symptoms: - Memory usage appears to grow but isn't a real leak - Profiling shows unexpected allocations

    Solutions: - Run garbage collection before taking snapshots - Take multiple snapshots to identify real trends - Focus on long-term growth rather than short-term fluctuations - Use production-like data volumes for profiling

    Issue: Transaction State is Confusing¶

    Symptoms: - Can't tell if in a transaction - Unexpected rollbacks or commits

    Solutions: - Add transaction state logging - Create a transaction context inspector - Use database-specific transaction monitoring queries - Implement transaction nesting counters
