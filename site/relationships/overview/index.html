    Relationships in Ormax¶

    Relationships are a fundamental aspect of database modeling that allow you to connect different data entities. Ormax provides a comprehensive system for defining and working with relationships between models.

    Understanding Relationships¶

    In Ormax, relationships are primarily implemented through ForeignKeyField, which creates a link between two models. There are three main types of relationships:

    1. One-to-Many: One instance of Model A can be related to multiple instances of Model B
    2. Many-to-One: Multiple instances of Model A can be related to one instance of Model B
    3. Many-to-Many: Multiple instances of Model A can be related to multiple instances of Model B

    Note: Currently, Ormax fully supports One-to-Many and Many-to-One relationships through ForeignKeyField. Many-to-Many relationships are planned for future releases.

    Defining Relationships¶

    ForeignKeyField¶

    The primary way to define relationships in Ormax is using ForeignKeyField:

    from ormax.fields import ForeignKeyField

class Author(ormax.Model):
    name = CharField(max_length=100)
    bio = CharField(max_length=500, null=True)

class Book(ormax.Model):
    title = CharField(max_length=200)
    pages = IntegerField()
    author = ForeignKeyField(Author, related_name="books")

    This creates a relationship where: - Each Book has one Author - Each Author can have multiple Book instances (accessed via author.books)

    ForeignKeyField Options¶

    Option Description Default
    to The related model class or name Required
    related_name Name for reverse relationship modelname_set
    on_delete Behavior when referenced object is deleted CASCADE
    null Whether the field can be null True
    db_constraint Whether to create database constraint True

    on_delete Options¶

    Option Description
    CASCADE Delete the object when referenced object is deleted
    SET_NULL Set to NULL when referenced object is deleted (requires null=True)
    RESTRICT Prevent deletion of referenced object
    DO_NOTHING Take no action 
    # Example with custom options
class Book(ormax.Model):
    title = CharField(max_length=200)
    author = ForeignKeyField(
        Author, 
        related_name="books",
        on_delete="SET_NULL",
        null=True
    )

    Working with Relationships¶

    Forward Relationships¶

    Forward relationships are accessed directly through the field:

    # Get a book
book = await Book.objects().get(id=1)

# Access the author (forward relationship)
author = await book.author  # Returns Author instance or None

# Set a new author
new_author = await Author.get(name="New Author")
book.author = new_author
await book.save()

    Reverse Relationships¶

    Reverse relationships are accessed through the related manager:

    # Get an author
author = await Author.objects().get(id=1)

# Get all books by this author (reverse relationship)
books = await author.books.all()

# Create a new book for this author
new_book = await author.books.create(
    title="New Book", 
    pages=200
)

# Add an existing book to this author
existing_book = await Book.get(title="Existing Book")
await author.books.add(existing_book)

# Remove a book from this author
await author.books.remove(existing_book)

# Clear all books from this author
await author.books.clear()

    Relationship Managers¶

    Ormax provides specialized managers for working with relationships:

    RelatedManager (Forward)¶

    Handles forward relationships (the author field in the Book model):

    • get(instance): Get the related object
    • set(instance, related_instance): Set the related object

    ReverseRelatedManager (Reverse)¶

    Handles reverse relationships (the books manager in the Author model):

    • all(instance): Get all related objects
    • create(instance, **kwargs): Create a new related object
    • add(instance, *related_instances): Add existing objects
    • remove(instance, *related_instances): Remove relationships
    • clear(instance): Clear all relationships

    Advanced Relationship Patterns¶

    Self-Referential Relationships¶

    Models can have relationships to themselves:

    class Category(ormax.Model):
    name = CharField(max_length=100)
    parent = ForeignKeyField(
        "Category", 
        related_name="children",
        null=True
    )

    Multiple Relationships to Same Model¶

    You can have multiple relationships to the same model by using different related_name values:

    class Book(ormax.Model):
    title = CharField(max_length=200)
    author = ForeignKeyField(Author, related_name="written_books")
    editor = ForeignKeyField(Author, related_name="edited_books")

    Nullable Relationships¶

    When a relationship is optional, set null=True:

    class Book(ormax.Model):
    title = CharField(max_length=200)
    translator = ForeignKeyField(Author, related_name="translations", null=True)

    Querying Across Relationships¶

    Filtering by Related Fields¶

    # Books by authors with "Ahmad" in their name
books = await Book.objects().filter(author__name__contains="Ahmad").all()

# Books by a specific author
books = await Book.objects().filter(author__id=1).all()

    Selecting Related Objects¶

    To avoid N+1 query problems, use select_related:

    # Get books with authors in one query
books = await Book.objects().select_related("author").all()

# Now accessing book.author won't trigger additional queries
for book in books:
    print(book.author.name)  # No additional query

    Prefetching Related Collections¶

    For reverse relationships, use prefetch_related:

    # Get authors with their books
authors = await Author.objects().prefetch_related("books").all()

# Now accessing author.books.all() won't trigger additional queries per author
for author in authors:
    for book in author.books.all():  # No additional queries
        print(book.title)

    Combining Both Techniques¶

    # Complex query with both select_related and prefetch_related
books = await Book.objects() \
    .select_related("author") \
    .prefetch_related("reviews") \
    .all()

    Relationship Examples¶

    Blog Post and Author Example¶

    class Author(ormax.Model):
    name = CharField(max_length=100)
    email = EmailField(unique=True)
    bio = TextField(null=True)

class BlogPost(ormax.Model):
    title = CharField(max_length=200)
    content = TextField()
    author = ForeignKeyField(Author, related_name="posts")
    published = BooleanField(default=False)
    published_date = DateTimeField(null=True)
    created_at = DateTimeField(auto_now_add=True)

async def create_post():
    # Create author
    author = await Author.create(
        name="Ahmad Mahmoudi",
        email="ahmad@example.com",
        bio="Technical writer and Python enthusiast"
    )

    # Create post
    post = await BlogPost.create(
        title="Introduction to Ormax ORM",
        content="Ormax is a powerful async ORM for Python...",
        author=author
    )

    # Get author's posts
    posts = await author.posts.all()
    print(f"{author.name} has {len(posts)} posts")

    # Update post
    post.published = True
    post.published_date = datetime.now()
    await post.save()

    # Get published posts
    published_posts = await BlogPost.objects().filter(published=True).all()
    print(f"There are {len(published_posts)} published posts")

    E-commerce Product and Category Example¶

    class Category(ormax.Model):
    name = CharField(max_length=100, unique=True)
    description = TextField(null=True)
    parent = ForeignKeyField("Category", related_name="children", null=True)

class Product(ormax.Model):
    name = CharField(max_length=200)
    description = TextField(null=True)
    price = DecimalField(max_digits=10, decimal_places=2)
    stock = IntegerField(default=0)
    category = ForeignKeyField(Category, related_name="products")
    created_at = DateTimeField(auto_now_add=True)

async def setup_store():
    # Create categories
    electronics = await Category.create(name="Electronics")
    phones = await Category.create(
        name="Mobile Phones", 
        parent=electronics
    )
    laptops = await Category.create(
        name="Laptops", 
        parent=electronics
    )

    # Create products
    await Product.create(
        name="Smartphone X", 
        price=999.99, 
        stock=50,
        category=phones
    )
    await Product.create(
        name="Laptop Pro", 
        price=1499.99, 
        stock=30,
        category=laptops
    )

    # Get all electronics products
    electronics_products = await Product.objects().filter(
        category__in=[phones, laptops]
    ).all()
    print(f"There are {len(electronics_products)} electronics products")

    # Get category hierarchy
    root_categories = await Category.objects().filter(parent=None).all()
    for category in root_categories:
        print(f"Category: {category.name}")
        children = await category.children.all()
        for child in children:
            print(f"  Subcategory: {child.name}")
            products = await child.products.all()
            for product in products:
                print(f"    Product: {product.name} (${product.price})")

    Best Practices for Relationships¶

    Naming Conventions¶

    • Use singular names for forward relationships (author, not authors)
    • Use plural names for reverse relationships (books, not book)

    Performance Considerations¶

    • Always use select_related for single-valued relationships (ForeignKey)
    • Use prefetch_related for multi-valued relationships (reverse ForeignKey)
    • Be careful with nested prefetching as it can lead to complex queries

    Database Design¶

    • Add indexes to foreign key fields for better performance
    • Consider whether relationships should be nullable
    • Think carefully about on_delete behavior based on your business logic

    Troubleshooting Common Issues¶

    Related Object Not Loading¶

    Issue: book.author returns a coroutine object instead of the author

    Solution: Remember to await the relationship: 

    author = await book.author  # Correct
author = book.author        # Incorrect - returns coroutine

    Reverse Relationship Name Conflict¶

    Issue: AttributeError: 'Author' object has no attribute 'book_set'

    Solution: Specify a custom related_name: 

    class Book(ormax.Model):
    author = ForeignKeyField(Author, related_name="books")

    MultipleObjectsReturned with Reverse Relationships¶

    Issue: await author.books.all() returns more objects than expected

    Solution: Check your filters and ensure you're not accidentally creating duplicate relationships

    Foreign Key Constraint Violation¶

    Issue: IntegrityError: FOREIGN KEY constraint failed

    Solution: - Ensure the related object exists before creating the relationship - Verify you're using the correct primary key value - Check if the relationship should be nullable
