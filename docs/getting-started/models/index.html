    Basic Model Definition¶

    Models are the core of any ORM system. In Ormax, models represent database tables and provide an intuitive Python interface to interact with your data. This guide will show you how to define and work with models in Ormax.

    Defining Your First Model¶

    Creating a model in Ormax is as simple as subclassing ormax.Model and declaring your fields:

    import ormax
from ormax.fields import CharField, IntegerField

class Book(ormax.Model):
    title = CharField(max_length=200)
    pages = IntegerField()

    This creates a model with two fields: - title: A string field with maximum length of 200 characters - pages: An integer field for page count

    Registering Models with Database¶

    Before you can use your model, you need to register it with your database instance:

    # Create database connection
db = ormax.Database("sqlite:///library.db")

# Register the model
db.register_model(Book)

    Creating Database Tables¶

    After registering your models, you need to create the corresponding database tables:

    # Create all registered tables
await db.create_tables()

# Or create a specific table
await db.create_table(Book)

    Field Types¶

    Ormax provides a rich set of field types for different data needs. Each field type corresponds to an appropriate database column type.

    String Fields¶

    CharField¶

    name = CharField(max_length=100, min_length=2)
    • Stores strings with a fixed maximum length
    • Automatically optimized for database-specific constraints (e.g., MySQL's 191 character limit for unique indexes)

    TextField¶

    content = TextField(max_length=5000, null=True)
    • For longer text content
    • No maximum length by default (uses database's TEXT type)

    EmailField¶

    email = EmailField()
    • Validates email format
    • Subclass of CharField with built-in validation

    URLField¶

    website = URLField()
    • Validates URL format
    • Subclass of CharField with built-in validation

    SlugField¶

    slug = SlugField(max_length=50)
    • Stores URL-friendly strings
    • Validates format (only letters, numbers, hyphens, and underscores)

    Numeric Fields¶

    IntegerField¶

    age = IntegerField(min_value=0, max_value=120)
    • Stores integer values
    • Supports min/max validation

    BigIntegerField¶

    population = BigIntegerField()
    • For very large integers

    SmallIntegerField¶

    rating = SmallIntegerField(min_value=1, max_value=5)
    • For smaller integer values (16-bit)

    FloatField¶

    price = FloatField(min_value=0.0)
    • Stores floating-point numbers

    DecimalField¶

    price = DecimalField(max_digits=10, decimal_places=2)
    • For precise decimal values
    • Ideal for financial data

    Date and Time Fields¶

    DateTimeField¶

    created_at = DateTimeField(auto_now_add=True, use_tz=True)
updated_at = DateTimeField(auto_now=True)
    • Comprehensive timezone support
    • auto_now_add: Set to current time when object is first created
    • auto_now: Set to current time on every save
    • use_tz: Whether to use timezone-aware datetime objects (default: True)

    DateField¶

    birth_date = DateField()
    • Stores just dates (without time)

    TimeField¶

    start_time = TimeField()
    • Stores just times (without date)

    Special Fields¶

    BooleanField¶

    is_active = BooleanField(default=True)
    • Accepts various formats (True/False, 1/0, "true"/"false", etc.)

    JSONField¶

    metadata = JSONField(null=True)
    • Stores JSON-serializable data
    • Automatically converts between Python objects and JSON

    UUIDField¶

    id = UUIDField(primary_key=True)
    • Stores UUID values
    • Automatically converts between string and UUID objects

    IPAddressField¶

    ip_address = IPAddressField(protocol="both")  # "ipv4", "ipv6", or "both"
    • Validates and stores IP addresses

    Auto Fields¶

    AutoField¶

    id = AutoField(primary_key=True)
    • Auto-incrementing integer field
    • Default primary key if none specified

    BigAutoField¶

    id = BigAutoField(primary_key=True)
    • Auto-incrementing big integer field

    Field Options¶

    All fields accept these common options:

    • primary_key: Whether this field is the primary key (default: False)
    • null: Whether the field can be null (default: True)
    • default: Default value for the field
    • unique: Whether the field must be unique (default: False)
    • index: Whether to create an index for this field (default: False)
    • help_text: Documentation for the field
    • db_column: Custom column name in the database (default: field name)

    Example with Multiple Options¶

    class Product(ormax.Model):
    sku = CharField(
        max_length=50,
        primary_key=True,
        help_text="Stock Keeping Unit - unique product identifier"
    )
    name = CharField(max_length=200, unique=True)
    description = TextField(null=True)
    price = DecimalField(max_digits=10, decimal_places=2, default=0.0)
    stock = IntegerField(default=0, min_value=0)
    created_at = DateTimeField(auto_now_add=True)
    is_active = BooleanField(default=True)

    Model Meta Options¶

    You can customize model behavior using the _meta attribute:

    class Book(ormax.Model):
    title = CharField(max_length=200)

    _meta = {
        "table_name": "library_books"  # Custom table name
    }

    Available Meta Options¶

    • table_name: Custom name for the database table (default: model class name in lowercase)
    • indexes: List of custom indexes to create

    Example with Multiple Meta Options¶

    class User(ormax.Model):
    username = CharField(max_length=50, unique=True)
    email = CharField(max_length=100, unique=True)

    _meta = {
        "table_name": "auth_users",
        "indexes": [
            {"fields": ["username", "email"], "unique": True}
        ]
    }

    Working with Models¶

    Creating Model Instances¶

    Method 1: Using the constructor¶

    # Create a new book instance
book = Book(title="شیر و خورشید", pages=350)

# Save to database
await book.save()

    Method 2: Using create() (saves automatically)¶

    # Create and save in one step
book = await Book.create(title="هویج", pages=200)

    Accessing Field Values¶

    # Get field values
print(book.title)  # "شیر و خورشید"
print(book.pages)  # 350

# Set field values
book.pages = 360
await book.save()

    Deleting Models¶

    # Delete a model instance
await book.delete()

    Converting to Dictionary¶

    # Convert model to dictionary
book_dict = book.to_dict()
print(book_dict)
# {'title': 'شیر و خورشید', 'pages': 350, 'id': 1}

# Create model from dictionary
new_book = Book.from_dict(book_dict)
await new_book.save()

    Advanced Model Features¶

    Custom Primary Keys¶

    By default, Ormax adds an auto-incrementing integer primary key named id. You can override this:

    class Product(ormax.Model):
    sku = CharField(max_length=50, primary_key=True)
    name = CharField(max_length=200)

    Default Values¶

    class BlogPost(ormax.Model):
    title = CharField(max_length=200)
    content = TextField()
    created_at = DateTimeField(default=datetime.now)
    is_published = BooleanField(default=False)

    Validation¶

    Ormax automatically validates data based on field types and constraints:

    # This will raise a ValidationError
book = Book(title="Very Long Title That Exceeds The Maximum Length", pages=350)
await book.save()  # ValidationError: Value exceeds maximum length of 200

    Model Relationships¶

    Ormax provides robust support for database relationships. Let's define a simple author-book relationship:

    ForeignKeyField¶

    from ormax.fields import ForeignKeyField

class Author(ormax.Model):
    name = CharField(max_length=100)
    bio = CharField(max_length=500, null=True)

class Book(ormax.Model):
    title = CharField(max_length=200)
    pages = IntegerField()
    author = ForeignKeyField(Author, related_name="books")

    Working with Relationships¶

    Forward Relationship¶

    # Get book's author
author = await book.author

# Set book's author
book.author = new_author
await book.save()

    Reverse Relationship¶

    # Get all books by an author
books = await author.books.all()

# Create a new book for the author
new_book = await author.books.create(
    title="New Book", 
    pages=200
)

# Add existing book to author
await author.books.add(existing_book)

    Complete Example¶

    Here's a complete example showing model definition and basic usage:

    import ormax
from ormax.fields import (
    CharField, 
    IntegerField, 
    DateTimeField,
    ForeignKeyField
)
import asyncio

# Create database connection
db = ormax.Database("sqlite:///library.db")

# Define models
class Author(ormax.Model):
    name = CharField(max_length=100)
    bio = CharField(max_length=500, null=True)
    created_at = DateTimeField(auto_now_add=True)

class Book(ormax.Model):
    title = CharField(max_length=200)
    pages = IntegerField(min_value=1)
    author = ForeignKeyField(Author, related_name="books")
    published_date = DateTimeField(null=True)

    _meta = {
        "table_name": "library_books"
    }

# Register models
db.register_model(Author)
db.register_model(Book)

async def main():
    # Create tables
    await db.create_tables()

    # Create authors
    ahmad = await Author.create(name="احمد محمودی", bio="نویسنده رمان‌های اجتماعی")
    saman = await Author.create(name="سامان سلطانی", bio="نویسنده کودک و نوجوان")

    # Create books
    await Book.create(title="شیر و خورشید", pages=350, author=ahmad, published_date="2020-01-15")
    await Book.create(title="هویج", pages=200, author=saman, published_date="2021-03-22")

    # Query books
    all_books = await Book.objects().all()
    print(f"Total books: {len(all_books)}")

    # Filter books
    ahmad_books = await Book.objects().filter(author=ahmad).all()
    print(f"Ahmad's books: {len(ahmad_books)}")

    # Get author's books using reverse relationship
    ahmad = await Author.objects().get(name="احمد محمودی")
    ahmad_books = await ahmad.books.all()
    print(f"Ahmad's books via reverse relationship: {len(ahmad_books)}")

    # Update a book
    book = ahmad_books[0]
    book.pages = 360
    await book.save()

    # Delete a book
    await ahmad_books[1].delete()

# Run the example
asyncio.run(main())

    Best Practices¶

    Naming Conventions¶

    • Use singular model names (Book instead of Books)
    • Use lowercase with underscores for field names (first_name not firstName)

    Field Design¶

    • Be explicit about nullability (null=True or null=False)
    • Use appropriate field types for your data
    • Add validation constraints where needed

    Performance Considerations¶

    • Use TextField instead of CharField for long text
    • Add indexes to frequently queried fields
    • Consider using defer() or only() for large models

    Troubleshooting¶

    Common Issues¶

    Field Not Found¶

    Error: DatabaseError: Field 'xyz' does not exist on model 'MyModel'

    Solution: Check that the field name is correctly spelled and registered in your model.

    Invalid Value¶

    Error: ValidationError: Value exceeds maximum length of 100

    Solution: Ensure your data meets the field constraints (length, type, etc.).

    Database Table Not Created¶

    Error: OperationalError: no such table: mymodel

    Solution: Make sure you've called await db.create_tables() after registering your models.
